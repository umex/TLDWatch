<objective>
Fix the SC-4 vram indicator defect that 02-UAT.md diagnosed as major:
`probe_vram`'s two CPU error-fallbacks (vram.py:149-155 psutil-import-fail,
vram.py:167-173 psutil-call-fail) return `loaded=[]` instead of
`loaded=_loaded_list(manager_state)`. Every other backend's fallbacks preserve
`loaded`. The defect was triggered live because `psutil` is declared in
pyproject.toml but was not installed in the runtime env, so `import psutil`
raised -> CPU fallback -> `loaded=[]` immediately after a 200 load.

Purpose: Restore the WR-03 "what's in VRAM" indicator contract on the CPU path
and lock it with a live-behavior test (the existing 155-test suite only
asserted `loaded==[]` + the shape — the test-coverage gap the UAT calls out).

Output: CPU error-fallbacks aligned with the other backends, psutil installed
in the runtime env, and a new test that loads a model then asserts
/diagnostics/vram reflects it on CPU, including a psutil-absent variant.
</objective>
