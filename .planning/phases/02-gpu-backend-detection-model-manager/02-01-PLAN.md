# Plan 02-01 — First-run GPU detect + burn-in test + settings.json wire-in

## Phase Goal

**As a** user on either machine, **I want to** drop in any video without configuring which GPU I have, **so that** the back-end silently picks CUDA / ROCm / CPU and persists it for every later component to read.

## Objective

Silently detect the active GPU backend (CUDA / ROCm / CPU) on first boot, run a real-kernel burn test to confirm the path actually uses the GPU (not silent CPU fallback, Pitfall 1 / 12), and persist the result to `data/settings.json` atomically using the Phase-1 D-04 atomic-write helper. Extend `Settings` and `UpdateSettingsRequest` with the seven Phase 2 fields so the on-disk schema carries everything the model manager and the future settings panel will read (D-08 declare-now). Add three back-end endpoints (`POST /diagnostics/gpu-burn`, `GET /diagnostics/vram`, `POST /diagnostics/test-hf-token`) that the future settings panel uses. The HF token is base64-encoded on disk and never returned in `GET /settings` (D-05). The detect never refuses to start — a CPU fallback is silent log-only (D-06).

**Purpose:** SC-1 fully delivered here. SC-2's preset table is declared as enum/field types here; the values that fill `PRESETS` live in 02-02's `app/models/presets.py`.

**Output:**
- `app/models/diagnostics.py` — `BackendProbe`, `VRAMState`, `LoadedModelInfo`, `GpuBurnResult`, `HfTokenResult`, `ModelSpec`, `ModelSet`, `GpuBackend`, `QualityPreset`, `ModelCategory`
- `app/models/backend.py` — `GpuBackend` enum + `detect()` + `burn_test()` functions
- `app/models/vram.py` — `probe_vram()` + `ManagerState`
- `app/models/hf_token.py` — `validate_token()` (HF Hub HEAD shim, no pyannote.audio import)

### New module `app/models/backend.py`
- Module-level `_log = logging.getLogger(__name__)` (the `app.jobs.service.py:37` pattern).
- `async def detect() -> GpuBackend` — Stage 1 cheap probe. Ordered checks: (1) `subprocess.run(["pip", "show", "torch"], timeout=3, capture_output=True, text=True)` — parse the `Version:` line for `+cu` or `+rocm`; (2) `subprocess.run(["nvidia-smi", "--query-gpu=name,memory.total", "--format=csv"], timeout=3, ...)`; (3) `os.environ.get("ROCM_PATH") or os.environ.get("HIP_PATH")`; (4) `import torch` (lazy) + `torch.cuda.is_available()` + `torch.cuda.device_count()` + `torch.version.hip`; (5) `subprocess.run(["powershell", "-Command", "Get-WmiObject Win32_VideoController | Select-Object Name"], timeout=3, ...)` for the device name. Returns `GpuBackend.CUDA` if `+cu` wheel + nvidia-smi works + a torch device is visible; returns `GpuBackend.ROCM` if `+rocm` wheel + a torch device is visible (HIP is `torch.cuda` API); returns `GpuBackend.CPU` otherwise (D-06 silent fallback). All `subprocess.run` calls wrapped in `try/except (subprocess.TimeoutExpired, FileNotFoundError, OSError)` — a missing `nvidia-smi` is "not present," not an error.
- `async def burn_test(backend: GpuBackend) -> BackendProbe` — Stage 2 real-kernel run. Per backend:
  - `CUDA` / `ROCM` (HIP): `import torch` (lazy); `a = torch.randn(1024, 1024, device="cuda"); b = torch.randn(1024, 1024, device="cuda"); torch.cuda.synchronize(); t0 = time.perf_counter(); (a @ b).item(); torch.cuda.synchronize(); t1 = time.perf_counter()`; record `burn_test_ms = (t1 - t0) * 1000`. Read `device_name = torch.cuda.get_device_name(0)` and `vram_total_mb = int(torch.cuda.mem_get_info(0)[1] / 1024**2)`. If `burn_test_ms > 5000`, log a WARN ("kernel took >5s; may have fallen back to CPU"); the probe is still returned (the fallback to CPU is the user's call, not ours — D-06).
  - `CPU`: `device_name = "CPU"`, `driver_version = None`, `vram_total_mb = None`, `burn_test_ms = None`, `notes = "no GPU detected; running in CPU mode"`.
  - Returns a complete `BackendProbe` with `probed_at = utcnow_iso()`.
- The function is `async def` (mirroring `app.jobs.reconcile_all`) so the lifespan can `await` it. The torch calls are synchronous inside (no real async torch API).

### New module `app/models/vram.py`
- `class ManagerState(BaseModel)` — `model_config = ConfigDict(extra="forbid")`; `live_vram_bytes: dict[ModelCategory, int] = Field(default_factory=dict)`. Module-level singleton `_manager_state: ManagerState = ManagerState()`.
- `def get_manager_state() -> ManagerState` — accessor (02-02's `configure_manager` swaps via `set_manager_state`).
- `def set_manager_state(state: ManagerState) -> None` — installed by `configure_manager` so `GET /diagnostics/vram` sees the live state.
- `def probe_vram(backend: GpuBackend, manager_state: ManagerState) -> VRAMState` — `ManagerState` is the typed holder. Per backend:
  - `CUDA` / `ROCM`: lazy `import torch`; `if not torch.cuda.is_available(): return VRAMState(backend=backend, total_mb=0, available_mb=0, used_mb=0, loaded=[])`; `free, total = torch.cuda.mem_get_info(0)`; `torch_alloc = torch.cuda.memory_allocated(0)`; `llm_pool = sum(manager_state.live_vram_bytes.values())` (the two-pool fix, Pitfall 2); `used = torch_alloc + llm_pool`; `available = free - llm_pool`; return `VRAMState` with `total_mb = int(total / 1024**2)`, `available_mb = int(available / 1024**2)`, `used_mb = int(used / 1024**2)`, `loaded` populated from the `ManagerState`.
  - `CPU`: lazy `import psutil`; `ps = psutil.virtual_memory()`; return `VRAMState(backend=backend, total_mb=int(ps.total / 1024**2), available_mb=int(ps.available / 1024**2), used_mb=int(psutil.Process(os.getpid()).memory_info().rss / 1024**2), loaded=[])`.
- `import torch` and `import psutil` are INSIDE the function body (NOT at module top), so a CPU-only test does not crash on a missing torch. This is the same lazy-import discipline `app.storage.db` uses for the engine listener.
- For 02-01, the `loaded` list is built from `manager_state.live_vram_bytes` with a placeholder `model_id = category.value + ":unknown"` and `loaded_at = utcnow_iso()`; 02-02 plumbs the real `model_id` and `loaded_at` from `ManagerState.loaded_meta`.

### New module `app/api/routes_diagnostics.py`
- `router = APIRouter(prefix="/diagnostics", tags=["diagnostics"])`.
- `@router.post("/gpu-burn", response_model=GpuBurnResult)` — calls `await backend_module.detect()`, `await backend_module.burn_test(backend)`, builds a new `Settings` via `current().model_copy(update={"backend": backend, "backend_probe": probe})`, writes atomically via the Phase-1 atomic-write helper + swaps in-memory via `settings_service.configure`. Returns `GpuBurnResult(probe=probe, active_backend=backend, settings_written=True)`. The route does NOT call `apply_update` (the detect result is a back-end-initiated write, not a user PATCH). The re-detect is a hot-swap — no `X-Restart-Required` header (H1: only `data_dir` is restart-required).
- `@router.get("/vram", response_model=VRAMState)` — calls `probe_vram(current().backend, get_manager_state())`, returns the state.
- `@router.post("/test-hf-token", response_model=HfTokenResult)` — reads `current().hf_token`, calls `await validate_token(token, repo_id="pyannote/speaker-diarization-3.1")`, returns the result. The helper's contract is "never raise, always return a typed result" (Pitfall 3); the route returns 200 for `skipped` (including network errors).
- The router is registered in `app/main.py` via `app.include_router(diagnostics_router)`.
