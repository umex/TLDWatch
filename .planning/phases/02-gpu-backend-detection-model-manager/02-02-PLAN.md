# Plan 02-02 ã Model manager: download, verify, lazy load, idle unload, VRAM probe, model API

## Phase Goal

**As a** user on either machine, **I want to** have my models downloaded on-demand and resident in VRAM one at a time, **so that** the 8 GB laptop never OOMs and the desktop can opt into a larger model without code changes.

## Objective

Build the `ModelManager` that owns the lifecycle of every model file on disk and in VRAM. The manager downloads models via `huggingface_hub` (resumable via the library's built-in `<blob>.incomplete` mechanism), verifies size and SHA256, lazy-loads them with an 85% VRAM budget gate, refuses concurrent residency unless the user opts in (D-04: refuse with 409, auto-swap is a Phase 4 follow-up), unloads explicitly (D-03: no time-based timer), and emits a structured per-model VRAM log line at load time. Add the `GET /models` + `POST /models/{id}/{download,status,load,unload}` API surface. Wire the manager singleton in the lifespan and surface its `ManagerState.live_vram_bytes` to the `GET /diagnostics/vram` endpoint from 02-01. Phase 2 ships the download/load MECHANISM; the prefetch-at-submit POLICY is a Phase 4 orchestrator concern (D-02 just-in-time at stage start).

**Purpose:** SC-2, SC-3, SC-4, SC-5 fully delivered here. The default model set (D-09 BALANCED) is the typed data behind the future settings panel; the download log API + the resume guarantee + the SHA verify are SC-3; the VRAM budget gate + the unload endpoint + the diagnostics indicator are SC-4; the `concurrent_models: bool` opt-in is SC-5 (D-04 refuse-then-caller-unloads).

**Output:**
- `app/models/manager.py` ã the `ModelManager` class + 5 typed error classes + the in-memory singleton + structured logging
- `app/models/registry.py` ã the `REGISTRY` dict that maps short ids to `ModelSpec`
- `app/models/presets.py` ã the `PRESETS` dict + `active_model_set(settings)` resolver (D-09)
- `app/storage/models_dir.py` ã `data_models_dir(settings)` + `ensure_models_dir(settings)` + category subdir helpers
- `app/api/routes_models.py` ã 6 new routes (GET, POST download, GET status, GET SSE, POST load, POST unload)
- Extended `app/models/vram.py` ã `ManagerState.loaded_meta` field + `set_manager_state`
- Extended `app/main.py` lifespan: `configure_manager(ModelManager(...))` after the session factory; `unload_all` on teardown
- 4 new test files + extended `tests/conftest.py` with `mock_hf_hub_download`, `mock_probe_vram` (extended), `configured_model_manager` fixtures

### New module `app/models/manager.py`
- Module-level `_log = logging.getLogger(__name__)`.
- Typed error hierarchy (5 classes):
  - `class ModelManagerError(Exception)` ã base.
  - `class VramBudgetExceeded(ModelManagerError)` ã `__init__(self, category: ModelCategory, needed_mb: int, available_mb: int)`; route maps to 507.
  - `class ConcurrentModelRefused(ModelManagerError)` ã `__init__(self, loaded_category: ModelCategory, requested_category: ModelCategory)`; route maps to 409 (D-04).
  - `class ModelGatedError(ModelManagerError)` ã `__init__(self, repo_id: str)`; route maps to 403 with the `"add HF token in settings"` fix.
  - `class ModelIntegrityError(ModelManagerError)` ã `__init__(self, repo_id: str, expected_sha: str, got_sha: str)`; route maps to 500.
- `class LoadedModel(BaseModel)` ã `model_config = ConfigDict(extra="forbid")`; fields: `category: ModelCategory`, `model_id: str`, `spec: ModelSpec`, `vram_bytes: int`, `loaded_at: str` (ISO).
- `class DownloadProgress(BaseModel)` ã fields: `model_id: str`, `bytes_done: int`, `bytes_total: int | None`, `state: Literal["queued", "running", "verifying", "done", "failed", "resuming"]`, `message: str | None = None`.
- `class ModelManager`:
  - `def __init__(self, settings: Settings, settings_factory: Callable[[], Settings] | None = None)`: store `_settings = settings`, `_state = ManagerState()`, `_settings_factory = settings_factory or (lambda: current())`. The factory lets the manager re-read settings on every load call (a PATCH to `vram_budget_fraction` or `concurrent_models` is picked up without a restart ã H1 hot-swap).
  - `async def ensure_downloaded(self, spec: ModelSpec, category: ModelCategory) -> Path` ã lazy-import `huggingface_hub.hf_hub_download` and the error classes (`GatedRepoError`, `RepositoryNotFoundError`) inside the function body. Resolve the target via `spec_file_path(self._settings, category, spec)`. Implement the size + SHA fast-path: if `target.exists() and spec.expected_size_bytes and target.stat().st_size == spec.expected_size_bytes`, return the target. If `target.exists() and spec.expected_sha256` and the SHA does not match, delete the file (corrupt; re-download). Call `hf_hub_download(repo_id=spec.repo_id, filename=spec.file or "model.bin", revision=spec.revision or "main", local_dir=target.parent, token=_get_token())` (the `_get_token` helper reads `current().hf_token` and decodes the base64-encoded token via the field_validator from 02-01 ã D-05). Wrap the call in `try/except (GatedRepoError, RepositoryNotFoundError)`: re-raise as `ModelGatedError(spec.repo_id)` and `ModelManagerError` respectively. After the download, if `spec.expected_sha256` is set, compute the file's SHA and raise `ModelIntegrityError` on mismatch (bounded retry: 1 re-download, no infinite loop ã Pitfall 4). Return the target path. This is the D-01 on-demand per-category download path.
  - `async def load(self, category: ModelCategory, spec: ModelSpec) -> LoadedModel` ã re-read settings via the factory. Check concurrent policy: `live = list(self._state.live_vram_bytes.keys())`; if `len(live) > 0 and not settings.concurrent_models`, raise `ConcurrentModelRefused(loaded_category=live[0], requested_category=category)` (D-04). Estimate model size: `expected_mb = (spec.expected_size_bytes or 0) / 1024**2`; multiply by 1.2 for LLM, 1.5 for STT (the per-category overhead from RESEARCH.md). Probe VRAM via `probe_vram(settings.backend, self._state)` (Pitfall 2 two-pool fix). Check budget: `budget_mb = state.total_mb * settings.vram_budget_fraction`; if `state.used_mb + expected_mb > budget_mb`, raise `VramBudgetExceeded(category, needed_mb=int(expected_mb), available_mb=int(budget_mb - state.used_mb))`. Record: `self._state.live_vram_bytes[category] = int(expected_mb * 1024**2)`; build `LoadedModel(category=category, model_id=spec.repo_id, spec=spec, vram_bytes=int(expected_mb * 1024**2), loaded_at=utcnow_iso())` and store in `self._state.loaded_meta[category]`. Emit the structured log line via `_log.info(json.dumps({...}))` (the keys from the must-have). Return the `LoadedModel`. **NOTE:** Phase 2 does NOT instantiate a faster-whisper or llama.cpp model ã the load is a typed VRAM reservation. The real weight loading happens inside the Phase 3 / Phase 8 adapters; the manager owns the lifecycle (reservation + unload), the adapters own the inference.
