# Plan 02-02 — Model manager: download, verify, lazy load, idle unload, VRAM probe, diagnostics endpoints

## Objective

Build the `ModelManager` that owns the lifecycle of every model file on disk and in VRAM. The manager downloads models via `huggingface_hub` (resumable via the library's built-in `<blob>.incomplete` mechanism), verifies size and SHA256, lazy-loads them with an 85% VRAM budget gate, refuses concurrent residency unless the user opts in, and emits a structured per-model VRAM log line at load time. Add the `GET /models` + `POST /models/{id}/{download,status,load,unload}` API surface that the future settings panel and the Phase 4 orchestrator will call. Wire the manager singleton in the lifespan and surface its `ManagerState.live_vram_bytes` to the `GET /diagnostics/vram` endpoint from 02-01.

**Purpose:** SC-2, SC-3, SC-4, SC-5 are fully delivered here. The default model set (BALANCED preset) is the typed data behind the future settings panel; the download log API + the resume guarantee + the SHA verify are SC-3; the VRAM budget gate + the unload endpoint + the diagnostics indicator are SC-4; the `concurrent_models: bool` opt-in is SC-5.

**Output:**
- `app/models/manager.py` — the `ModelManager` class + 4 typed error classes + the in-memory singleton + structured logging
- `app/models/registry.py` — the `REGISTRY` dict that maps short ids to `ModelSpec` (the canonical manifest)
- `app/models/presets.py` — the `PRESETS` dict + `active_model_set(settings)` resolver
- `app/storage/models_dir.py` — `data_models_dir(settings)` + `ensure_models_dir(settings)` + category subdir helpers
- `app/api/routes_models.py` — 6 new routes (GET, POST download, GET status, GET SSE, POST load, POST unload)
- Extended `app/main.py` lifespan: `configure_manager(ModelManager(...))` after the session factory
- Extended `pyproject.toml` with `faster-whisper>=1.0` (just the import, not the model weight; the manager test mocks the actual `WhisperModel` constructor)
- 4 new test files + extended `tests/conftest.py` with a `mock_hf_hub_download` fixture and a `mock_probe_vram` fixture

### New module `app/models/manager.py`
- Module-level `_log = logging.getLogger(__name__)`.
- Typed error classes (no Phase 1 analog; small class hierarchy):
  - `class ModelManagerError(Exception)` — base.
  - `class VramBudgetExceeded(ModelManagerError)` — `__init__(self, category: ModelCategory, needed_mb: int, available_mb: int)`; the route layer maps to 507.
  - `class ConcurrentModelRefused(ModelManagerError)` — `__init__(self, loaded_category: ModelCategory, requested_category: ModelCategory)`; the route layer maps to 409.
  - `class ModelGatedError(ModelManagerError)` — `__init__(self, repo_id: str)`; the route layer maps to 403 with the `"add HF token in settings"` fix.
  - `class ModelIntegrityError(ModelManagerError)` — `__init__(self, repo_id: str, expected_sha: str, got_sha: str)`; the route layer maps to 500.
- `class LoadedModel` — a `BaseModel` (typed result, not a Protocol) with fields: `category: ModelCategory`, `model_id: str`, `spec: ModelSpec`, `vram_bytes: int`, `loaded_at: str` (ISO). The `Protocol` shape from RESEARCH.md is relaxed to a Pydantic model for testability and OpenAPI exposure.
- `class DownloadProgress` — a `BaseModel` with fields: `model_id: str`, `bytes_done: int`, `bytes_total: int | None`, `state: Literal["queued", "running", "verifying", "done", "failed", "resuming"]`, `message: str | None = None`. (Mirrors RESEARCH.md §"Download log API".)
- `class ModelManager`:
  - `def __init__(self, settings: Settings, settings_factory: Callable[[], Settings] | None = None)`: stores the settings + a factory for hot-swap re-reads (the manager may outlive a PATCH /settings change to `vram_budget_fraction`; the factory lets the manager re-read on every load call). The factory defaults to `lambda: current()` (the `app/settings/service.py` accessor).
  - `async def ensure_downloaded(self, spec: ModelSpec, category: ModelCategory) -> Path` — returns the on-disk path. The flow:
    1. Resolve the target path via `spec_file_path(settings, category, spec)`.
    2. If the target exists and `spec.expected_size_bytes` is set and `target.stat().st_size == spec.expected_size_bytes`, return the target (size match, skip SHA on the happy path).
    3. If `spec.expected_sha256` is set and the file exists but its SHA does not match, delete the file (corrupt; re-download).
    4. Call `huggingface_hub.hf_hub_download(repo_id=spec.repo_id, filename=spec.file or "model.bin", revision=spec.revision or "main", local_dir=target.parent, token=current().hf_token)` (the `token` is read from settings at call time; the lazy import is inside the function body for test seam).
    5. If `spec.expected_sha256` is set, compute the SHA of the downloaded file; if it does not match, raise `ModelIntegrityError`.
    6. Catch `huggingface_hub.errors.GatedRepoError` (lazy import for the exception class) and re-raise as `ModelGatedError(spec.repo_id)`.
    7. Catch `huggingface_hub.errors.RepositoryNotFoundError` and re-raise as `ModelManagerError` (unmapped; the route layer returns 500).
    8. Return the target path.
  - `async def load(self, category: ModelCategory, spec: ModelSpec) -> LoadedModel` — the central budget-and-concurrency gate. The flow:
    1. Re-read settings via the factory: `settings = self._settings_factory()`.
    2. Check concurrent policy: `live = list(self._state.live_vram_bytes.keys())`; if `len(live) > 0 and not settings.concurrent_models`, raise `ConcurrentModelRefused(loaded_category=live[0], requested_category=category)`.
    3. Estimate model size: `expected_mb = (spec.expected_size_bytes or 0) / 1024**2`. Add 20% overhead for LLM, 50% overhead for STT (the STACK.md budget math from RESEARCH.md).
    4. Probe VRAM: `state = probe_vram(settings.backend, self._state)`.
    5. Check budget: `budget_mb = state.total_mb * settings.vram_budget_fraction`; if `state.used_mb + expected_mb > budget_mb`, raise `VramBudgetExceeded(category, needed_mb=int(expected_mb), available_mb=int(budget_mb - state.used_mb))`.
    6. Record the allocation: `self._state.live_vram_bytes[category] = int(expected_mb * 1024**2)`.
    7. Build and return `LoadedModel(category=category, model_id=spec.repo_id, spec=spec, vram_bytes=int(expected_mb * 1024**2), loaded_at=utcnow_iso())`.
    8. Emit the structured log line: `_log.info(json.dumps({"event": "model_loaded", "category": category.value, "model_id": spec.repo_id, "expected_vram_mb": int(expected_mb), "measured_vram_mb_after_load": int(state.used_mb + expected_mb), "total_vram_mb": state.total_mb, "available_vram_mb_after_load": int(state.total_mb - state.used_mb - expected_mb)}))`. The line is JSON-shaped so a future diagnostics panel can parse it. (The `caplog` test captures the line by `caplog.records[i].message` and parses the JSON.)
    **Note:** Phase 2 does NOT actually instantiate a faster-whisper or llama.cpp model — the load is a typed reservation. The real model construction is Phase 3 (STT) and Phase 8 (LLM). The manager reserves VRAM, emits the log, and returns the `LoadedModel`. The Phase 3 / Phase 8 adapters will read the `LoadedModel` to know "the user has loaded the STT model; run inference on it" — but the actual weight loading happens inside the adapter. This is the right separation: the manager owns the lifecycle (reservation + unload); the adapters own the inference.
  - `async def unload(self, category: ModelCategory) -> None` — idempotent. If `category in self._state.live_vram_bytes`, delete the entry; emit the structured log line `{"event": "model_unloaded", "category": category.value}`. Return `None` either way.
  - `async def unload_all(self) -> None` — for the lifespan teardown. Clears `self._state.live_vram_bytes`; emits one log line per cleared category.
  - `def currently_loaded(self) -> list[LoadedModel]` — returns a list of `LoadedModel` reconstructed from `self._state.live_vram_bytes` and the parallel `self._state.loaded_meta: dict[ModelCategory, LoadedModel]` dict (updated in `load` / `unload`).
  - `async def verify(self, spec: ModelSpec) -> bool` — walks the file, computes the SHA, compares to `spec.expected_sha256`. Returns `True` if the SHA matches or `spec.expected_sha256 is None` (no declared SHA; pass). Returns `False` on mismatch.
  - `def list_installed(self) -> list[ModelSpec]` — walks `data_models_dir(settings)` and returns the `ModelSpec`s whose target files exist on disk. Uses the `REGISTRY` (every spec is potentially installed) and filters by `target.exists()`. The `category` is recovered from the file's position in the tree.
- Module-level singleton: `_manager: ModelManager | None = None` + `def get_manager() -> ModelManager` (raises `RuntimeError` if not configured, mirroring `app/settings/service.py:144-147`) + `def configure_manager(manager: ModelManager) -> None` (used by the lifespan).
- The `ManagerState` from 02-01 is extended in this plan: add `loaded_meta: dict[ModelCategory, LoadedModel] = Field(default_factory=dict)`. The `app/models/vram.py` module re-exports the extended type. `get_manager_state()` returns the manager's state (a module-level pointer that `configure_manager` updates).

### New module `app/api/routes_models.py`
- `router = APIRouter(prefix="/models", tags=["models"])`.
- Six routes (all `Depends(get_settings)` for consistency with the rest of the API; the manager is read via `get_manager()`):
  1. `GET /models` — `response_model=ModelsListResponse` (`{"installed": list[ModelSpec], "available": list[tuple[str, ModelSpec]], "active_set": ModelSet}`). Calls `manager.list_installed()` and `list_specs()`; computes `active_set = active_model_set(settings)`. The route is `async def`. The `(id, spec)` pairs in `available` are flattened to a list of `{"id": id, "spec": spec}` for the OpenAPI shape.
  2. `POST /models/{id}/download` — `response_model=DownloadTaskResponse` (`{"task_id": str, "status_url": str}`), status code 202. Resolves the spec via `registry.get_spec(id)`, kicks off an `asyncio.create_task(self._run_download(spec, category, task_id))`, returns the task id and the status URL.
