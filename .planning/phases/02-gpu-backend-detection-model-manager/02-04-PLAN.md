<objective>
Fix the SC-3 download defect that 02-UAT.md diagnosed as a blocker: the synchronous
`huggingface_hub.hf_hub_download` is awaited directly inside the async background task
`_run_download`, freezing the event loop for the entire download. This single defect
breaks WR-01 (409 duplicate-in-flight), WR-02 (live SSE heartbeat + byte-level progress),
and HW-09 (resume-after-crash via the Xet backend does not use the .incomplete + Range
path the code assumes).

Purpose: Restore the live download UX contract that the mocked 155-test suite could not
catch (the test-coverage gap the UAT explicitly calls out).

Output: Thread-offloaded download path, classic non-Xet resume forced, and new
live-behavior tests that assert 409 + live SSE + resume without mocking the download
away.
</objective>
