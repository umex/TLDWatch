<objective>
Close UAT gap (05-UAT.md test 5, entry 5 — MINOR): a completed job's history row shows duration `--:--` while old failed jobs show `00:42`. The completed job's `duration_s` is not rendering.

Diagnosed root cause: the orchestrator's happy path NEVER populates `duration_s`. The `transcribed` transition (orchestrator.py:290-294) passes `ManifestPatch(language=transcript.language)` with NO `duration_s`, and the `done` transition (orchestrator.py:295-296) passes no patch at all. `update_stage` re-projects `new_manifest.duration_s` (manifest.py:244) — which stays `None` because nothing ever sets it. So `GET /jobs/{id}` returns `duration_s: null` and `HistoryRow.tsx:49` renders `formatDuration(null) -> "--:--"`. The old failed jobs showing `00:42` had `duration_s` set via a different path (test fixtures / a prior probe); the current happy path leaves it blank.

The chunker ALREADY computes the source media duration at chunker.py:130 (`total_seconds = total_samples / SAMPLE_RATE`) but does not propagate it through the returned `Transcript` (the model has no `duration_s` field). The cleanest, semantically correct fix: add an optional `duration_s` field to `Transcript`, populate it with `total_seconds` on both chunker return paths (fast path line ~160, chunked path line ~252), and pass it into the manifest via the `transcribed` transition's `ManifestPatch(language=..., duration_s=transcript.duration_s)`. The `done` transition then re-projects the already-set value (no patch needed on done). `duration_s` becomes the source MEDIA duration — matching the field's colocation with `source_sha256`/`source_path` (media metadata) and the `00:42` the user expects.

Purpose: Completed jobs show their media duration in the history list, matching the failed-jobs display.
Output: Transcript.duration_s field, chunker population on both paths, orchestrator transcribed-transition projection, an orchestrator test asserting duration_s is non-null on done, a HistoryRow test asserting a non-blank duration renders.
</objective>
