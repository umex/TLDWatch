<objective>
Back-end ingest stage: the streaming upload endpoint and transcript read endpoint the front-end consumes, plus the pre-queued `'uploading'` status that prevents the Phase 4 worker from picking up a job mid-upload.

Purpose: This is the "other half" of Phase 4 D-04's generalized `ingested` check — the browser cannot supply a server-side `source_path`, so Phase 5 adds a streaming route that writes `data/jobs/<id>/source.<ext>` atomically and enqueues. The transcript read endpoint (D-14) serves the parsed Phase 3 `Transcript` so the detail view can render it. Together these close INGEST-01 (back-end) and the back-end half of JOB-03 (history list + re-open).

Output: `POST /jobs/upload`, `GET /jobs/{id}/transcript`, the `'uploading'` JobStatus literal, the widened `enqueue` clause, the `create_upload_job` helper, and 7 passing back-end test files.
</objective>
