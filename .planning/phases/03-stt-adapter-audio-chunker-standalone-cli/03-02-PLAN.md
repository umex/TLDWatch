<objective>
Implement the windowed audio chunker with OOM halve-and-retry and midpoint-overlap stitch (D-02, INGEST-05). The chunker is a pure-Python orchestrator over the STTAdapter Protocol (D-06) — it imports NEITHER faster_whisper NOR ctranslate2 at module top; the only faster_whisper touch is a lazy in-body import of decode_audio to decode the file once into a numpy array that is then sliced per chunk. This plan delivers the long-audio safety valve that SC-2 and INGEST-05 mandate.

Purpose: int8_float16 (D-04) is the first OOM defense; the chunker is the second layer — the deterministic retry knob SC-2 requires. Without it, a 2-3 hour video can OOM on the 8 GB laptop with no recovery.
Output: app/models/stt/chunker.py, tests/test_chunker.py (green), full suite green.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| on-disk audio file → chunker.decode_audio | user-supplied local file path is decoded by PyAV (via the adapter) into a numpy array; the chunker slices the array |
| per-chunk transcribe → CTranslate2 OOM | the OOM RuntimeError crosses back into the chunker's retry loop |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03 | Tampering | malicious audio → PyAV parser during decode_audio | accept | PyAV bundles a fixed FFmpeg build (D-01); decode_audio wraps av.error.InvalidDataError. Input is user-supplied local files (trusted single-user per PROJECT.md). The chunker adds no new attack surface — it slices a numpy array. |
| T-03-04 | Denial of Service | OOM loop never terminates | mitigate | FLOOR_SECONDS = 60 hard floor; after the floor the final attempt is allowed to raise (no catch). The halving is `chunk_s //= 2` so the loop is bounded by log2(WINDOW/FLOOR) ≈ 4 iterations per chunk. Non-OOM RuntimeErrors re-raise immediately (Pitfall 5) so a non-OOM error cannot masquerade as OOM and loop forever. |
| T-03-SC | Tampering | (no new pip installs in this plan) | accept | This plan adds no packages — it reuses the 03-01 pins and the existing test infra. |

No high-severity threats. No V2/V3/V4/V6 categories apply.
