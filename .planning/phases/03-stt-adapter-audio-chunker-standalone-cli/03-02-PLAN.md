`def _transcribe_chunk_oom_safe(adapter, audio_slice, *, language, chunk_s) -> SttTranscription` (Pattern 3, REVISED per Codex HIGH — split-both-halves recursive retry, full coverage):
- If `chunk_s < FLOOR_SECONDS`: make one final attempt at the current slice length and let any RuntimeError raise (do not catch — below the floor, an OOM is a real failure, not a retryable one). Call `adapter.transcribe(audio_slice[: int(chunk_s * SAMPLE_RATE)], language=language, vad_filter=True, condition_on_previous_text=False)` and return the result.
- Otherwise try `adapter.transcribe(audio_slice[: int(chunk_s * SAMPLE_RATE)], language=language, vad_filter=True, condition_on_previous_text=False)`.
- On `RuntimeError as exc`: if `not _OOM_RE.search(str(exc))`: raise (Pitfall 5 — non-OOM RuntimeError re-raised unchanged, NOT split).   - `left = _transcribe_chunk_oom_safe(adapter, audio_slice[:mid], language=language, chunk_s=chunk_s / 2)`
  - `right = _transcribe_chunk_oom_safe(adapter, audio_slice[mid:], language=language, chunk_s=chunk_s / 2)`

  - This is the Codex HIGH fix: BOTH halves of a failed window are transcribed (recursively sub-chunking each half until it succeeds), so the full chunk duration is covered with no dropped remainder. The recursion is bounded: each OOM halves chunk_s, and below FLOOR_SECONDS (60 s) the final attempt raises (no further split) — so the depth is log2(WINDOW/FLOOR) ≈ 4 and the call tree is finite. ## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03 | Tampering | malicious audio → PyAV parser during decode_audio | accept | PyAV bundles a fixed FFmpeg build (D-01); decode_audio wraps av.error.InvalidDataError. Input is user-supplied local files (trusted single-user per PROJECT.md). The chunker adds no new attack surface — it slices a numpy array. |
| T-03-04 | Denial of Service | OOM split loop never terminates | mitigate | FLOOR_SECONDS = 60 hard floor; below the floor the final attempt is allowed to raise (no catch, no further split). The split-both-halves recursion halves chunk_s on each OOM and stops splitting below the floor, so the depth is bounded by log2(WINDOW/FLOOR) ≈ 4 and the call tree is finite. Non-OOM RuntimeErrors re-raise immediately (Pitfall 5) so a non-OOM error cannot masquerade as OOM and loop forever. The full-decode memory cost is ~115 MB/hour (documented); streaming decode is a future enhancement if multi-hour files become problematic. |
| T-03-SC | Tampering | (no new pip installs in this plan) | accept | This plan adds no packages — it reuses the 03-01 pins and the existing test infra. |
