<objective>
Build the orchestrator's state-machine driver, the in-process asyncio EventBus, and the kw-only `progress_cb`/`cancel_flag` extension to `STTAdapter.transcribe` + `transcribe_file`. This is the first slice of the Phase 4 spine: a job can be driven `queued -> ingesting -> transcribing -> done` with file-as-truth transitions and per-chunk progress published to the bus. Implements D-01 (local-only ingest), D-02 (no mid-chunk resume), D-04 (reference in place), D-08 (in-process bus), D-09 (per-chunk progress), and the load-bearing threading decision (threading.Event cancel_flag + call_soon_threadsafe progress_cb). Covers SC-1, JOB-02.

Purpose: Every later phase reduces to "add a stage" driven by this same loop. The bus + state machine here are the contracts 04-02 (worker/queue/cancel) and 04-03 (WS subscriber/idempotency) build on.
Output: `app/jobs/orchestrator.py` (run_job), `app/jobs/progress.py` (EventBus), extended `STTAdapter` Protocol + chunker, refined `ingested` check, Wave 0 test scaffolding + state-machine/bus tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| asyncio loop <-> worker thread (run_in_executor) | sync chunker runs off-loop; cancel_flag + progress_cb cross this boundary |
| orchestrator <-> SQLite DB | stage transitions cross via update_stage (manifest-first/DB-last) |
| orchestrator <-> EventBus | progress events published from call_soon_threadsafe |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
