<objective>
Build the orchestrator's state-machine driver, the in-process asyncio EventBus, the kw-only `progress_cb`/`cancel_flag` extension to `STTAdapter.transcribe` + `transcribe_file` + the concrete `FasterWhisperAdapter`, the shared `JobCancelled` exception, heartbeat + progress-snapshot persistence, and the `infer_resume_point` stage-skip integration. This is the first slice of the Phase 4 spine: a job can be driven `queued -> ingesting -> transcribing -> done` with file-as-truth transitions, stage completion recorded only after the output file exists, per-chunk progress + heartbeat published to the bus, and re-entrant correctness on restart.

Purpose: Every later phase reduces to "add a stage" driven by this same loop. The bus + state machine here are the contracts 04-02 (worker/queue/cancel) and 04-03 (WS subscriber/idempotency) build on. The review fixes folded in here (Fix 2, 3, 4, 5, 8, 9-partial) close operational correctness gaps surfaced by Codex + Ollama: no false-stale kills on long transcriptions, no TypeError in production, no upward STT->orchestration import, no early stage completion, no missed wakeup on partial failure, and nonzero progress for reconnecting WS clients.

Output: `app/jobs/errors.py` (JobCancelled), `app/jobs/orchestrator.py` (run_job with infer_resume_point + functools.partial + graceful shutdown + heartbeat + progress.json), `app/jobs/progress.py` (EventBus), extended `STTAdapter` Protocol + chunker + FasterWhisperAdapter, refined `ingested` check, `app/storage/fs.py` (`_STAGE_FILE_NAMES` adds `progress.json` — Fix 2 root cause), Wave 0 test scaffolding + state-machine/bus tests.
