# Plan 01-03 — Per-job filesystem layout, stage files, resume rule, cleanup helpers

## Objective

Complete the per-job filesystem story. After this plan:
- Every stage file from D-03 has a typed path helper + atomic write helper + read helper: `transcript.json`, `diarization.json`, `summary-<kind>.json`, `edits.json`, plus the existing `manifest.json` and `source.<ext>`
- The full `JobManifest` (matching the Pydantic model from Plan 01-01) can be round-tripped to/from the manifest file with `read_manifest` and `write_manifest`
- The resume rule (D-12) is implemented as `infer_resume_point(manifest) -> StageName | None`: walks the files in the standard order and returns the first incomplete stage, or `None` if all done
- Cleanup helpers exist: `cancel_job` (delete folder + mark DB cancelled), `mark_failed` (mark DB failed, keep folder), `mark_stale` (mark DB failed if no stage write in 10 minutes — hardcoded per D-13)
- The `POST /jobs/{id}/cancel` route calls `cancel_job`; the `POST /jobs/{id}/stale-check` admin/test route calls `mark_stale` (Phase 4 owns the actual periodic checker)
- A helper `update_stage(settings, job_id, stage, manifest_fields=None)` updates `current_stage`, `stage_timestamps`, and `updated_at` in both the DB row and the manifest file atomically — every stage mutator in later phases uses this single helper (pitfall 9 mitigation)
- All new file paths in `data/jobs/<id>/` go through helpers in `app/storage/fs.py`; no other module constructs a `Path` into a job folder by string concatenation

This plan is the file-as-truth contract that the rest of the project relies on. Phase 4 (orchestrator) and every stage adapter call these helpers and never touch `Path("data/jobs")` directly.

## must_haves (goal-backward verification)

- [ ] **truth: complete stage file helpers** — `app/storage/fs.py` exports `transcript_path(settings, job_id) -> Path`, `diarization_path(settings, job_id) -> Path`, `summary_path(settings, job_id, kind) -> Path`, `edits_path(settings, job_id) -> Path`, `source_path(settings, job_id, ext) -> Path` (where `ext` is the user-provided source extension); all return paths inside `data/jobs/<id>/`
- [ ] **truth: manifest round-trip** — after `POST /jobs` (which writes the manifest), `read_manifest(settings, job_id)` returns a `JobManifest` whose `model_dump()` equals the JSON on disk; writing the manifest with one extra field and re-reading preserves that field
- [ ] **truth: resume rule (D-12)** — given a folder with `source.mp4` but no `transcript.json`, `infer_resume_point(...)` returns `"transcribed"` (the first incomplete stage); given all stage files present, returns `None`; given an unparseable stage file, logs a warning and returns that stage (re-run is the right behavior)
- [ ] **truth: cancel deletes the folder** — after `POST /jobs/{id}/cancel`, the `data/jobs/<id>/` directory is gone and the DB row's `status` is `"cancelled"`; subsequent `GET /jobs/{id}` returns 200 (the row stays) with `status="cancelled"`
- [ ] **truth: failure keeps the folder** — after `mark_failed(job_id, "some reason")`, the folder is intact and the DB row's `status` is `"failed"`, `error="some reason"`
- [ ] **truth: staleness uses 10 minutes** — `mark_stale(job_id, threshold_s=600)` marks the job `failed` with `error="stalled"` iff the last stage-file mtime is older than `threshold_s`; otherwise it is a no-op
- [ ] **truth: stage updates go through one helper** — `update_stage(...)` writes the new `current_stage` and `stage_timestamps[<stage>]` to BOTH the manifest file (atomically) and the DB row in a single transaction; subsequent `GET /jobs/{id}` reflects the new `current_stage`
- [ ] **verification**: `pytest -q` passes; `uvicorn app.main:app` boots; `curl -sf -X POST http://127.0.0.1:8000/jobs -H "Content-Type: application/json" -d '{}'` returns 201; the new resume + cleanup helpers can be invoked from a one-shot script and the folder reflects the expected state

## Artifacts this phase produces

### New functions in `app/storage/fs.py`
- `transcript_path(settings, job_id) -> Path` — returns `job_dir(settings, job_id) / "transcript.json"`
- `diarization_path(settings, job_id) -> Path` — returns `job_dir(settings, job_id) / "diarization.json"`
- `summary_path(settings, job_id, kind: str) -> Path` — returns `job_dir(settings, job_id) / f"summary-{kind}.json"`
- `edits_path(settings, job_id) -> Path` — returns `job_dir(settings, job_id) / "edits.json"`
- `source_path(settings, job_id, ext: str) -> Path` — returns `job_dir(settings, job_id) / f"source.{ext.lstrip('.')}"`
- `async def list_stage_files(settings, job_id) -> list[Path]` — returns the absolute paths of all stage files that currently exist for the job
- `def last_stage_mtime(settings, job_id) -> float | None` — returns the maximum mtime across all stage files in the job folder, or `None` if no files exist (used by the staleness check)

### New functions in `app/jobs/manifest.py`
- `async def read_manifest(settings, job_id) -> JobManifest` — reads the manifest file, validates via `JobManifest.model_validate_json`, returns the model
- `def manifest_mtime(settings, job_id) -> float | None` — returns the manifest file's mtime, or `None` if missing (for staleness checks that should include the manifest itself)
- `async def update_stage(settings, session, job_id, stage: str, manifest_patch: dict | None = None) -> JobManifest` — load current manifest (404 if missing), apply the patch, set `current_stage=stage`, set `stage_timestamps[stage] = datetime.utcnow().isoformat()+"Z"`, write manifest atomically, and in the same flow update the `jobs` row's `current_stage`, `stage_timestamps_json`, and `updated_at`. Returns the new manifest.

### New module `app/jobs/resume.py`
- `StageName = Literal["ingested", "transcribed", "diarized", "summarized", "done"]`
- `STAGE_ORDER: tuple[StageName, ...] = ("ingested", "transcribed", "diarized", "summarized", "done")` — the standard walk order from D-12
- `def is_stage_complete(stage: StageName, settings, job_id, manifest: JobManifest) -> bool` — for each stage, check the file-existence rule from D-11:
  - `ingested`: `source_path(...)` exists
  - `transcribed`: `transcript_path(...)` exists
  - `diarized`: `diarization_path(...)` exists (optional stage; if `manifest.diarization_enabled` is False, treat as complete; if missing for a long time, mark as skipped)
  - `summarized`: for every kind in `manifest.summary_kinds`, `summary_path(..., kind)` exists
  - `done`: terminal
- `def infer_resume_point(settings, job_id, manifest: JobManifest) -> StageName | None` — walk `STAGE_ORDER`, return the first stage where `is_stage_complete(stage, ...)` is False; return `None` if all complete
- `def parse_stage_file(path: Path) -> bool` — opens the file, attempts to parse it as JSON (or the appropriate format for source files — for `source.*` always returns True if the file exists and is non-empty); returns True on success, False on any error. Used by D-12 to mark unparseable files for re-run.

### New module `app/jobs/cleanup.py`
- `async def cancel_job(session, settings, job_id) -> bool` — if the job folder exists, recursively delete it; UPDATE the jobs row SET status='cancelled'; commit; return True if the row existed, False otherwise
- `async def mark_failed(session, job_id, error: str) -> bool` — UPDATE the jobs row SET status='failed', error=:error, updated_at=:now; commit; return True if the row existed
- `def is_stale(settings, job_id, threshold_s: int = 600) -> bool` — returns True iff the last stage-file mtime is older than `threshold_s` (or no stage files at all and the job is older than threshold_s)
- `async def mark_stale(session, settings, job_id, threshold_s: int = 600) -> bool` — if `is_stale(...)`, call `mark_failed(session, job_id, "stalled")`; return True if marked, False if not stale

### New routes in `app/api/routes_jobs.py`
- `POST /jobs/{id}/cancel` — calls `cancel_job`; returns 200 with the updated `JobResponse`, 404 if the job does not exist
- `POST /jobs/{id}/stage` — body: `StageUpdateRequest` (strict, fields: `stage: StageName`, optional `manifest_patch: dict | None = None`); calls `update_stage`; returns 200 with the new `JobManifest`. (Phase 4 orchestrator + every stage adapter calls this. It is exposed now because every later phase needs it.)
- `POST /jobs/{id}/stale-check` — body: `StaleCheckRequest` (strict, fields: `threshold_s: int = 600`); calls `mark_stale`; returns 200 with `{"marked": true|false, "stale": true|false}`. This is an admin/test endpoint in Phase 1; Phase 4 replaces the periodic call with its own background loop.

### New Pydantic models
- `app/models/job.py::StageUpdateRequest` — strict; fields: `stage: StageName` (import `StageName` from `app.jobs.resume`), `manifest_patch: dict[str, Any] | None = None` (typed loosely for Phase 1 — Phase 3+ narrows it to typed field updates)
- `app/models/job.py::StaleCheckRequest` — strict; fields: `threshold_s: int = 600`
- `app/models/job.py::StaleCheckResponse` — lax; fields: `stale: bool`, `marked: bool`

### Test files
- `tests/test_manifest_helpers.py` — write_manifest then read_manifest round-trip; missing manifest raises a known error (not a generic Exception)
- `tests/test_stage_files.py` — for each path helper, assert the path is inside `data/jobs/<id>/` and has the expected filename
- `tests/test_resume.py` — `infer_resume_point` with no files returns `"ingested"`; with `source.mp4` returns `"transcribed"`; with `source.mp4` + `transcript.json` returns `"diarized"`; with all stage files (matching `manifest.summary_kinds=[]`) returns `None`; with one summary kind missing returns `"summarized"`
- `tests/test_cleanup.py` — `cancel_job` deletes the folder and marks the DB row `cancelled`; `mark_failed` keeps the folder and marks the row `failed` with the given error; `mark_stale` with a 0-second threshold marks the row `failed (stalled)`; `mark_stale` with a 1-hour threshold is a no-op for a freshly-touched folder
